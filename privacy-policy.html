HTTP Status - 隐私政策

最后更新日期：2026年2月16日

简要说明：HTTP Status 扩展不收集、不存储、不传输任何用户个人信息。所有数据仅在您的浏览器本地处理，我们无法访问您的任何数据。

1. 扩展用途

HTTP Status 是一个开发者工具扩展，专门用于在浏览器地址栏中显示当前网页的 HTTP 状态码。该扩展帮助开发者快速了解网页的响应状态、重定向链路和服务器信息。

2. 数据收集与使用

2.1 我们不收集的数据

HTTP Status 扩展 不收集、不存储、不传输 以下任何信息：

  • 个人身份信息（姓名、邮箱、电话等）
  • 浏览历史记录
  • 访问的网站地址（URL）
  • 网页内容
  • 表单数据或用户输入
  • Cookie 或认证信息
  • 地理位置信息
  • 任何可识别用户身份的数据

2.2 本地数据处理

扩展仅在您的浏览器本地处理以下技术数据，这些数据不会离开您的设备：

  • HTTP 状态码：当前网页的 HTTP 响应状态（如 200、404 等）
  • 响应头信息：HTTP 响应头（如 Content-Type、Server 等）
  • 重定向链路：页面重定向的跳转记录
  • 服务器 IP 地址：当前网站的服务器 IP（由浏览器提供）
  • 扩展设置：您的个人偏好设置（存储在本地）

3. 权限说明

HTTP Status 请求以下浏览器权限，仅用于实现核心功能：

3.1 webRequest（网络请求监听）

  • 用途：监听 HTTP 请求和响应，获取状态码和响应头信息
  • 范围：仅读取 HTTP 响应的元数据（状态码、响应头）
  • 限制：不读取、不修改网页内容或请求正文

3.2 storage（本地存储）

  • 用途：保存扩展的配置选项和用户偏好设置
  • 范围：仅存储扩展相关的设置数据（如显示选项、调试模式等）
  • 限制：数据完全存储在本地，不同步到任何服务器

3.3 tabs（标签页访问）

  • 用途：获取当前标签页的基本信息（ID、URL），以便正确显示对应的 HTTP 状态
  • 范围：仅读取标签页元数据
  • 限制：不读取标签页内容，不修改标签页

3.4 host_permissions（主机权限）

  • 用途：访问所有网站的 HTTP 响应信息
  • 说明：由于用户可能访问任何网站，扩展需要此权限才能在所有网站上工作
  • 限制：仅读取 HTTP 元数据，不访问网页内容

4. 数据共享

我们不与任何第三方共享数据。

由于扩展不收集任何数据，因此没有数据被共享、出售或传输给任何第三方机构、广告商或数据分析公司。

5. 数据安全

  • 所有数据处理完全在您的浏览器本地进行
  • 扩展不包含任何追踪代码或分析工具
  • 扩展不连接任何外部服务器
  • 扩展不使用任何第三方服务或 SDK

6. 用户权利

您可以随时：

  • 卸载扩展（所有本地数据将被自动删除）
  • 清除扩展存储的设置（在扩展设置中）
  • 撤销扩展的权限（在浏览器扩展管理页面）

7. 儿童隐私

本扩展不专门针对 13 岁以下儿童，也不故意收集儿童的任何信息。由于扩展不收集任何数据，因此适合所有年龄段用户使用。

8. 隐私政策变更

如果我们更新此隐私政策，将在此页面发布新版本，并更新"最后更新日期"。重大变更将通过扩展更新说明通知用户。

9. 联系我们

如果您对此隐私政策有任何疑问或建议，请通过以下方式联系我们：

  • GitHub Issues: https://github.com/solarhell/HTTP-Status-docs/issues

HTTP Status - Privacy Policy

Last Updated: February 16, 2026

TL;DR: HTTP Status extension does NOT collect, store, or transmit any personal user information. All data is processed locally in your browser, and we have no access to any of your data.

1. Extension Purpose

HTTP Status is a developer tool extension designed to display the HTTP status code of the current webpage in the browser address bar. This extension helps developers quickly understand the response status, redirect chains, and server information of web pages.

2. Data Collection and Use

2.1 Data We Do NOT Collect

HTTP Status extension does NOT collect, store, or transmit any of the following information:

  • Personally identifiable information (name, email, phone, etc.)
  • Browsing history
  • Visited website URLs
  • Web page content
  • Form data or user input
  • Cookies or authentication information
  • Geolocation data
  • Any data that can identify users

2.2 Local Data Processing

The extension only processes the following technical data locally in your browser. This data never leaves your device:

  • HTTP Status Codes: HTTP response status of current webpage (e.g., 200, 404)
  • Response Headers: HTTP response headers (e.g., Content-Type, Server)
  • Redirect Chains: Page redirect jump records
  • Server IP Address: Server IP of current website (provided by browser)
  • Extension Settings: Your personal preference settings (stored locally)

3. Permission Explanations

HTTP Status requests the following browser permissions, used solely for core functionality:

3.1 webRequest (Network Request Monitoring)

  • Purpose: Monitor HTTP requests and responses to retrieve status codes and response headers
  • Scope: Only reads HTTP response metadata (status codes, headers)
  • Limitation: Does not read or modify web page content or request bodies

3.2 storage (Local Storage)

  • Purpose: Save extension configuration options and user preferences
  • Scope: Only stores extension-related settings (e.g., display options, debug mode)
  • Limitation: Data is completely stored locally, not synced to any server

3.3 tabs (Tab Access)

  • Purpose: Get basic information of current tab (ID, URL) to correctly display corresponding HTTP status
  • Scope: Only reads tab metadata
  • Limitation: Does not read tab content, does not modify tabs

3.4 host_permissions (Host Permissions)

  • Purpose: Access HTTP response information from all websites
  • Explanation: Since users may visit any website, the extension needs this permission to work on all websites
  • Limitation: Only reads HTTP metadata, does not access web page content

4. Data Sharing

We do NOT share data with any third parties.

Since the extension does not collect any data, no data is shared, sold, or transmitted to any third-party organizations, advertisers, or data analytics companies.

5. Data Security

  • All data processing is completely local in your browser
  • Extension contains no tracking code or analytics tools
  • Extension does not connect to any external servers
  • Extension does not use any third-party services or SDKs

6. User Rights

You can at any time:

  • Uninstall the extension (all local data will be automatically deleted)
  • Clear extension-stored settings (in extension settings)
  • Revoke extension permissions (in browser extension management page)

7. Children's Privacy

This extension is not specifically directed at children under 13 and does not knowingly collect any information from children. Since the extension does not collect any data, it is suitable for users of all ages.

8. Privacy Policy Changes

If we update this privacy policy, we will post the new version on this page and update the "Last Updated" date. Significant changes will be communicated through extension update notes.

9. Contact Us

If you have any questions or suggestions about this privacy policy, please contact us through:

  • GitHub Issues: https://github.com/solarhell/HTTP-Status-docs/issues
